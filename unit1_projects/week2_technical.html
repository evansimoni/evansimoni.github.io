What does it mean to display inline vs inline blocks?

When you create a website you are essentially just arranging a bunch of boxes on the screen that can contain text or pictures. The web languages that comprise the basic structure of most websites, HTML and CSS, work together to enable coders to customize how a page looks. CSS allows you to instruct certain boxes (or elements) to have specific attributes. These attributes play into how a web browser will render the visual arrangement of the website. While there are countless attributes that affect a page's look, I will be focusing on explaing the distinction between two of them -- display: inline and display: inline-block.

There are two default ways that an element will behave in relation to the elements around it. First, there are inline elements, which sit side by side until they fill up the width of the page and then continue on the next line. The other type of element is a block element, which will sit on its own line and force elements around it onto other lines.

An elements attribute is set in the CSS. In the case of display it would look like:

element { display: value; }

When an element's display is 'inline' (either by declaration or by default), this means the element will be forced to position itself on the same line as surrounding elements. The divs below are displayed inline:

When an element's display is inline-block, this means the element will position itself on the same line as surrounding elements (space permitting) but will also stay contained within its declared block's properties. In other words, it will still look like a block but won't get squished up against the surrounding elements like inline elements do.
