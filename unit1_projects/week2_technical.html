What does it mean to display inline vs inline blocks?

When you create a website you are essentially just arranging a bunch of boxes on the screen that can contain text or pictures. The web languages that comprise the basic structure of most websites, HTML and CSS, work together to enable coders to customize how a page looks. CSS allows you to instruct certain boxes (or elements) to have specific attributes. These attributes play into how a web browser will render the visual arrangement of the website. While there are countless attributes that affect a page's look, I will be focusing on explaing the distinction between two of them -- "display: inline" and "display: inline-block".

There are two default values that determine how elements will behave in relation surrounding elements. First, there are inline elements, which sit side by side until they fill up the width of their containing element and then continue onto the next line. The other type of element is a block element, which will occupy its own line and force other elements around it onto subsequent lines.

An elements attribute is set in the CSS. Using the display property it looks like this:

element { display: value; }

When an element's display is 'inline' (either by declaration or by default), this means the element will be forced to position itself on the same line as surrounding elements. Inline-block positions surrounding elements on the same line similar to the inline display value. This is most useful for arranging elements whose default display property is block. When an element's display is inline-block, this lets the element display inline while maintaining its default block properties. In other words, it will still look like a block and won't get squished up against the surrounding elements like inline elements do (ahem, see divs above). In conclusion, inline-block is useful for positioning block level elements intact and positionally seperate from each other. Simply using inline on block-type elements will not allow them to maintain their integrity and they will be pushed up on and overlapped by surounding elements.
